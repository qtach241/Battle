Requirements

Java version 7. Java can be found at www.java.com/getjava.

How to use

Setting up

1. Open Steam and click on Library.
2. Right click on Dota 2, and select Properties.
3. Select Set Launch Options.
4. Add -override_vpk to the launch options.
5. Click Ok.

Running the program

Simply open Dota_Sound_Editor.jar!

Using Dota 2 Sound Editor

The Dota 2 Sound Editor allows you to modify sounds for items, voices, music and hero spells. To begin, select a tab from the top of the window. Then, if applicable, select an entry from the dropdown box and expand the list of sounds associated with a particular action. From there, you can view each sound for a hero spell, item, musical trigger or voiceover. For each sound, you can:

Replace
This allows you to change what sound plays when the selected action is triggered.

Revert
This restores the selected action to its default sound.

Revert All
  • Hero Spells: This restores all actions for to current hero to their default sounds.
  • Items: This restores all item actions to their default sound.
  • Music: This restores all music for the current category to its default song.
  • Voice: This restores all voices for the current unit/announcer to their default sounds.

Advanced/Basic
The Advanced button allows you to view the current selection's entire sound script as a tree. Currently, manual editing is not supported, but will be added in the future.
While in Advanced mode, Replace, Revert and Play Sound will only work on nodes that contain a path to a sound file.
Clicking the Basic button will return the editor to Basic mode.

Troubleshooting

I can't launch the sound editor!

  • Check to make sure that the Dota_Sound_Editor.jar is in the same folder as the lib folder.
  • Ensure that your Java is up to date.

    • My custom sounds aren't playing in game!

  • Make sure you have -override_vpk set, as described in Setting Up.
  • If you are replacing an item sound, ensure that it is the same file type (WAV or MP3) as the original, and that it matches the original's bitrate (1141 kbps for WAVs, 128 kbps for MP3s) and frequency (44.1KHz).
  • Check that your sound file is a .WAV or .MP3 file. If it is, try re-encoding it at a lower frequency or bitrate--it's possible that Dota 2 only supports sound files in certain encodings.
  • Check that your sound was copied successfully. The program copies custom sounds to /Steam/steamapps/common/dota 2 beta/dota/sound/custom.
  • Check that the script file was modified appropriately. For example, if you changed Abaddon's Death sound to wilhelm.wav, you would navigate to /Steam/steamapps/common/dota 2 beta/dota/scripts/game_sounds_heroes/ and open game_sounds_abaddon.txt. Under the Hero_Abbadon.Death section, you would check to see if the wave section was pointed to custom/abaddon/wilhelm.wav.
  • If all else fails, the nuclear solution that might work is to delete your sound.cache from /Steam/steamapps/common/dota 2 beta/dota/sound, and then Verify the Integrity of your Game Cache. To do this, open Steam -> right-click Dota 2 -> select Properties -> select the Local Files tab -> Click Verify Integrity of Game Cache, and wait for it to complete. Afterward, try replacing the game sounds again.

    • My custom item sounds are being cut off!

    This seems to be a limitation of item sound modding. Custom item sounds must be the same length as, or shorter than, the sounds they are replacing. Sorry!

    Some of my regular game sounds aren't playing!

    Try verifying Dota's game cache. To do this, open Steam -> right-click Dota 2 -> select Properties -> select the Local Files tab -> Click Verify Integrity of Game Cache, and wait for it to complete.

    Alternately, this can happen if Valve adds a new sound for an existing action. To fix this, just open the sound editor, and select the Hero/Item/Music from the dropdown box. When the program loads those sounds, it will automatically update with any new sounds Valve has added.

    Wait, why does that happen?

    The sound editor copies the script files that describe what sounds to play from the internal game files. It then uses those copied script files to hold all your modifications, and to tell the game which sounds to play. Unfortunately, if Valve updates these script files, your modified files won't have those changes, so the editor has to get a fresh, updated version from the internal game files. Until it does so, any new actions won't know what sound to play.

    Found a bug?

    Please let me know. When the program crashes, it attempts to generate an error.txt. Attaching these files to an email, and giving a detailed description of what you were doing when the program crashed both help a lot in determining the cause of a bug or crash. Thanks!
